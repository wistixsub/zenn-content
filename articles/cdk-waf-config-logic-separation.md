## はじめに

WAF を CDK で書くと、`CfnWebACL` のような L1 コンストラクトが中心になり、書く内容はほぼ CloudFormation テンプレートそのまま、ということがよくあります。正直なところ「CDK にしたメリットは薄いな」と感じる場面です。

そんな L1 中心の構成でも CDK（プログラミング言語）の良さを活かせるとすれば、ループで同じようなルールをまとめて生成したり、`if` で条件分岐したり、といった**言語の基本機能**の部分です。

今回はもう一歩進めて、出来上がる WebACL の**詳細設定（WebACL の名前・どのルールを入れるか・各ルールのアクション）を JSON ファイルに切り出し**、構築のためのロジック部分は固定化しました。今後見込まれる変更は **JSON ファイルだけ**で完結するように、コードの構成を考えています。

同じように「L1 で CloudFormation を書いているのと変わらないな」と感じている方の、IaC 運用の参考になれば幸いです。
